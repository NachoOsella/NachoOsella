## Engineering taste

```txt
01. Model the domain first; let frameworks stay at the edges.
02. Prefer simple contracts over clever abstractions.
03. Make tests explain behavior, not implementation details.
04. Keep deployments reproducible and dependency surfaces small.
05. Write documentation that reduces future questions.
```
